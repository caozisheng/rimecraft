# 网页游戏Agent对话式开发工具 设计方案
本方案深度参考 OpenCut 项目的 Monorepo 工程架构、开源协作模式与技术栈规范，基于 React+Tauri 技术栈，面向网页游戏对话式开发场景完成全链路设计，包含游戏引擎选型、架构设计、功能模块、Agent 系统与落地路径。

## 一、核心游戏引擎选型
结合「网页游戏原生支持、TypeScript 全链路兼容、React/Tauri 无缝集成、Agent 代码生成低幻觉」四大核心原则，确定主选引擎与备选方案，同时对齐 OpenCut 开源、轻量、开箱即用的产品理念。

### 首选引擎（分场景）
| 场景 | 核心引擎 | 核心选型理由 |
|------|----------|--------------|
| 2D 网页游戏（主力推荐） | **Phaser.js** | 1.  纯 Web 原生、10 年+开源维护，MIT 协议与 OpenCut 完全一致，无商业风险<br>2.  全 TypeScript 支持，内置场景管理、物理引擎、动画系统、输入处理等完整游戏能力，开箱即用，大幅降低 Agent 代码生成的拼接成本与幻觉问题<br>3.  官方提供数千个可运行示例，完美适配 RAG 知识库构建，Agent 生成代码准确率远超其他引擎<br>4.  原生支持 Canvas/WebGL 渲染，Tauri 打包后桌面端性能无损耗，同时兼容 Web 端、H5、小游戏多端部署<br>5.  官方原生兼容 React 等 40+ 前端框架，与项目技术栈 100% 匹配 |
| 3D 网页游戏 | **React Three Fiber (R3F) + Three.js** | 1.  完全基于 React 声明式语法开发，与项目技术栈开发范式统一，Agent 可直接生成 React 组件实现 3D 能力，无缝嵌入项目架构<br>2.  基于 Web 3D 领域绝对霸主 Three.js 生态，原生支持 WebGPU，2026 年生态成熟度、文档完善度无竞品可替代<br>3.  组件化开发模式与对话式 Agent 迭代逻辑高度匹配，支持增量修改、热更新实时预览，符合低代码对话开发的核心诉求<br>4.  开源 MIT 协议，无商业使用限制，Tauri 打包后可调用系统 GPU 资源，保证 3D 渲染性能 |

### 备选引擎
1.  **PixiJS**：极致轻量的 2D 渲染引擎，适合高度自定义的渲染场景，需自行搭配物理、场景管理等第三方库，仅推荐给有进阶定制需求的场景，Agent 开发复杂度较高。
2.  **Babylon.js**：全功能 3D 游戏引擎，TS 原生支持，适合重型 3D 游戏开发，与 React 集成顺滑度弱于 R3F。
3.  **Hex Engine**：React 风格的 2D 游戏引擎，API 设计贴合 React Hooks 范式，但生态成熟度、示例丰富度远低于 Phaser.js，不推荐作为主力选型。

## 四、核心功能模块设计
围绕「自然语言对话 → Agent 生成/修改游戏 → 实时预览调试 → 项目导出发布」的核心闭环，拆分六大核心模块，完全贴合对话式开发的用户体验。

### 1. 对话式 Agent 交互模块（核心）
- 多轮对话界面：左侧沉浸式对话栏，支持上下文记忆、对话历史管理、会话分支切换，用户可通过自然语言描述游戏需求、玩法、角色、交互逻辑
- 需求拆解与任务调度：Agent 自动识别用户需求，拆解为场景搭建、角色开发、交互逻辑、玩法实现、UI 开发等子任务，协调多智能体协同完成
- 工具调用能力：Agent 可自主调用文件管理、代码生成、素材检索、游戏运行时、错误调试等工具，实现端到端的开发流程，无需人工手动操作
- 双向代码同步：Agent 生成/修改的代码实时同步到代码编辑器，用户手动修改的代码也会实时同步到 Agent 上下文，保证开发一致性
- 自动错误修复：实时捕获游戏运行的报错、异常、性能问题，自动分析原因并修复代码，重新热更新到预览窗口

### 2. 游戏引擎运行时与实时预览模块
- 引擎标准化封装：对 Phaser.js/R3F 进行二次封装，提供标准化、低幻觉的 API 接口，统一游戏生命周期、场景管理、资源加载规范，保证 Agent 生成代码的可运行性
- 安全沙箱运行：基于 iframe + ESBuild 实现代码安全沙箱，隔离游戏运行环境，避免恶意代码执行，同时实现毫秒级热更新，代码修改后实时刷新预览
- 双分屏实时预览：右侧固定游戏预览面板，支持游戏画面实时渲染、暂停/继续、帧率监控、Draw Call 统计、控制台日志输出，支持多场景切换预览
- 资源热加载：图片、音频、精灵图、瓦片地图等素材修改后，无需重启游戏即可实时更新到运行时，适配对话式迭代开发的高频修改需求

### 3. 项目与文件管理模块
- 全生命周期项目管理：支持游戏项目的新建、打开、克隆、删除、导出，内置标准化的项目结构，Agent 可基于固定结构生成代码，避免结构混乱
- 可视化文件树：类 VS Code 文件树管理，支持文件/文件夹的新建、重命名、删除、拖拽，自动分类代码文件、素材文件、配置文件，支持全局文件检索
- 素材资源管理：支持图片、音频、精灵图、瓦片地图、字体等游戏素材的上传、预览、分类、标签检索，内置免费可商用的素材库，Agent 可自动匹配需求并生成引用代码
- 版本管理：原生集成 Git，支持项目的版本提交、回滚、分支管理，对齐 OpenCut 的开源协作模式，支持多人协作开发
- 多格式导出：一键导出 Web 部署包、Tauri 桌面端安装包、H5 小游戏包，支持发布到 itch.io、Steam、抖音小游戏等平台

### 4. 代码编辑器模块
- 基于 Monaco Editor 封装，与 VS Code 体验完全一致，支持 TypeScript 语法高亮、智能补全、错误提示、断点调试
- 集成 Biome 实现一键代码格式化、语法校验，配置规则与 OpenCut 完全对齐，保证代码风格统一
- 内置游戏开发代码片段库，覆盖 Phaser.js/R3F 常用 API 与玩法模板，支持 Agent 快速调用与用户手动插入
- 支持分屏编辑、多文件同时打开，适配复杂游戏项目的开发需求

### 5. 模板与辅助工具模块
- 游戏模板库：内置平台跳跃、RPG 冒险、射击、解谜、卡牌、休闲合成等主流游戏类型的完整模板，用户可一键创建，Agent 基于模板进行增量修改，大幅降低开发门槛
- 性能监控面板：实时显示游戏帧率、内存占用、Draw Call、资源加载耗时，Agent 可基于性能数据自动优化代码，降低渲染开销
- 内置知识库：集成 Phaser.js/R3F 官方文档、游戏开发最佳实践、API 手册，支持用户手动检索与 Agent RAG 增强检索
- 游戏测试工具：内置玩家操作模拟、碰撞检测调试、场景跳转工具，支持 Agent 自动生成测试用例，验证游戏功能完整性

## 五、Agent 系统核心设计
对话式开发的核心竞争力，采用多智能体协作架构，参考 OpenCut 隐私优先的理念，同时支持云端模型与本地离线模型。

### 1. 多智能体协作架构
| 智能体角色 | 核心职责 |
|------------|----------|
| 主控 Agent | 对话入口，负责解析用户需求、拆解任务、协调其他智能体工作，管理全局对话上下文与项目状态 |
| 游戏设计 Agent | 基于用户需求完成游戏玩法设计、场景规划、角色设定、交互逻辑设计、数值体系设计，输出标准化设计文档 |
| 代码生成 Agent | 基于设计文档，生成符合 Phaser.js/R3F 规范与项目结构的 TypeScript 代码，严格遵循编码规范与 API 最佳实践 |
| 调试修复 Agent | 捕获游戏运行报错、异常、性能问题，结合错误日志与代码上下文，定位问题并完成代码修复与优化 |
| 素材匹配 Agent | 根据游戏设计需求，检索内置素材库与用户上传素材，匹配对应的图片、音频、精灵图资源，生成合规的资源引用代码 |
| 测试 Agent | 自动生成游戏测试用例，模拟玩家操作，验证游戏玩法完整性、边界情况处理，输出测试报告与优化建议 |

### 2. 核心能力设计
- **RAG 增强检索**：构建 Phaser.js/R3F 官方文档、最佳实践、示例代码、开源项目的知识库，Agent 生成代码前先完成检索，从根源降低大模型幻觉，保证代码的可运行性与合规性
- **长上下文记忆**：支持超长上下文窗口，完整记住项目代码结构、用户历史需求、游戏设计逻辑，避免多轮迭代中出现前后矛盾、代码覆盖等问题
- **增量迭代开发**：支持用户对游戏进行持续的修改、优化、功能新增，Agent 基于历史上下文进行增量代码修改，而非全量重写，保证开发效率
- **全离线运行支持**：兼容 DeepSeek-R1、Llama 3、Qwen 等本地开源大模型，无需联网即可完成全流程开发，完全保护用户的游戏项目隐私，对齐 OpenCut「Your videos stay on your device」的隐私理念
- **严格的约束体系**：通过系统提示词与代码预校验，严格限制 Agent 生成符合 TypeScript 语法、引擎 API 规范、项目结构的代码，禁止生成无效代码、恶意代码、不符合规范的代码

## 六、开发与部署方案
完全对齐 OpenCut 的开发流程与部署模式，保证开发体验一致性，同时适配 Tauri 跨端特性。

### 1. 开发环境搭建
#### 前置依赖
- Node.js 20+、Bun 1.0+
- Rust 工具链（Tauri 开发必需）
- Docker + Docker Compose（可选，用于本地数据库与 Redis 服务）
- VS Code / Cursor 编辑器

#### 快速启动步骤（同 OpenCut 对齐）
1.  Fork 仓库并克隆到本地
2.  根目录执行 `bun install` 安装全量依赖
3.  复制环境变量模板：`cp .env.example .env.local`，配置 LLM API 密钥、数据库地址等必填参数
4.  启动数据库与缓存服务（可选）：`docker-compose up -d`
5.  执行数据库迁移：`cd apps/web && bun run db:migrate`（同 OpenCut 操作规范）
6.  启动全量开发服务：`bun dev`，Turborepo 自动启动所有应用
    - Web 端访问地址：http://localhost:3000
    - Tauri 桌面端自动启动应用窗口
    - 游戏预览服务地址：http://localhost:3001

### 2. 构建与部署方案
- **Web 端**：使用 Next.js 构建生产包，一键部署到 Vercel / Netlify，与 OpenCut 部署模式完全对齐，支持 CDN 加速、全球访问
- **桌面端**：使用 Tauri 2.x 构建，生成 Windows/macOS/Linux 全平台安装包，适配 Steam 上架与桌面端分发
- **移动端**：基于 Tauri 2.x 移动端能力，构建 iOS/Android 安装包，一套代码实现多端覆盖
- **私有化部署**：提供 Docker 一键部署镜像，支持企业/个人完全私有化、离线部署，无任何数据泄露风险

### 3. 开源协作规范
- 采用 MIT 开源协议，与 OpenCut 完全一致，无商业使用限制
- 明确贡献焦点区域：Agent 能力迭代、游戏引擎适配、UI/UX 优化、模板库扩充、Bug 修复
- 明确暂不接受的贡献：游戏渲染内核重构、导出功能重写（预留架构升级空间，同 OpenCut 规范）
- 提供完整的 CONTRIBUTING.md 贡献指南，规范开发流程、PR 提交规范、代码风格要求
